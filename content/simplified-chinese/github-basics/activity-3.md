# 房间 3：拉取请求传送门

{{< figure src="../images/room3_pull_request.jpg" title="一个巨大的门挡在你面前，上面写着‘拉取请求’" >}}

一个巨大的门挡住了你的去路，上面写着“拉取请求”。旁边的牌子写着：“要打开这扇门，您的更改必须经过审核。”

## 你的任务

1. 将本地更改推送到您分叉的仓库。
2. 向原始 `CodeDungeon/escape-room` 仓库创建拉取请求。

## 说明

1. 推送您的更改：

```
git push origin main
```

2. 前往您在 GitHub 上分叉的仓库。
3. 点击“贡献”，然后点击“打开拉取请求”。
4. 填写标题为“房间 2 的解决方案”，并提供简要描述。
5. 点击“创建拉取请求”。

{{< notice info >}}
**提示：** 密切关注您的拉取请求。地牢守护者（仓库维护者）会审核并可能要求更改！
{{< /notice >}}

{{< expand "拉取请求流程" >}}

**推送更改**
首先，将您的本地更改推送到您的远程仓库：

```
git push origin main
```

**创建拉取请求**
1. 前往您在 GitHub 上的分叉
2. 点击“Pull requests”选项卡
3. 点击“New pull request”
4. 确保基础仓库是原始的，而头部仓库是您的分叉
5. 点击“创建拉取请求”

**等待审核**
创建拉取请求后：
1. 等待维护者审核您的更改
2. 解决他们提供的任何反馈
3. 必要时推送新的提交以更新拉取请求
